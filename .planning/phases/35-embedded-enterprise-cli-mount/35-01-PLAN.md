# Plan 35-01: Add Embedded Commander Mount API

## Goal

Expose Atlas as a mountable Commander command tree so enterprise CLIs can call `attachAtlas(program, config)` and get `userCli <namespace> <all Atlas commands/options/flags>` without duplicating command definitions or adding unsupported schema fields.

## Objective

Executor will refactor current `createAtlasProgram(runtime)` into reusable command-tree construction, add constrained `AtlasMountConfig`, export `createAtlasCommand()` and `attachAtlas()`, preserve standalone binary behavior, and package the new subpath import.

## Scope

- Reuse existing command registration for standalone and mounted Atlas command.
- Add public mount API for Commander-based enterprise CLIs.
- Keep wrapper schema constrained to current supported fields only.
- Inject mount defaults through existing config/env/global option mechanisms.
- Preserve explicit CLI flags over wrapper defaults.
- Add package subpath export for `@moxellabs/atlas/commander`.

## Out of Scope

- No `logo`, `color`, `docsUrl`, `supportUrl`, `productName`, visual branding, auth hooks, token callbacks, or broad SDK.
- No new `--atlas-mcp-resource-prefix` flag or `ATLAS_MCP_RESOURCE_PREFIX` env var in this plan.
- No rewrite of command implementations under `apps/cli/src/commands/` except call-site adjustments required by shared runtime.

## must_haves

- `attachAtlas(program, config)` mounts namespace command under caller-provided Commander program.
- `createAtlasCommand(config)` returns a Commander `Command` named by `config.namespace`.
- Mounted namespace exposes same current Atlas subcommands/options/flags below namespace.
- Standalone `atlas` behavior remains compatible through `runCli()` and default `createAtlasProgram()` path.
- Public schema contains only `namespace`, `identityRoot`, `mcp.name`, `mcp.title`, `mcp.resourcePrefix`, and `defaults.config/cacheDir/logLevel/caCertPath`.
- Explicit user flags such as `userCli acme --atlas-identity-root .override search q` override wrapper defaults.
- Package build exports `@moxellabs/atlas/commander`.

<action>
Create a public Commander mount module, preferably `apps/cli/src/commander.ts`, exporting exactly these public types/functions:

```ts
import type { Command } from "commander";

export type AtlasMountConfig = {
  namespace: string;
  identityRoot?: string;
  mcp?: {
    name?: string;
    title?: string;
    resourcePrefix?: string;
  };
  defaults?: {
    config?: string;
    cacheDir?: string;
    logLevel?: "debug" | "info" | "warn" | "error";
    caCertPath?: string;
  };
};

export function createAtlasCommand(config: AtlasMountConfig): Command;
export function attachAtlas(
  program: Command,
  config: AtlasMountConfig,
): Command;
```

Do not add any public fields named `logo`, `color`, `docsUrl`, `supportUrl`, `productName`, `auth`, `token`, `tokenProvider`, or `authProvider`. Validate `namespace` as a non-empty Commander command segment with no whitespace. Validate `identityRoot`, `mcp.name`, and `mcp.resourcePrefix` by reusing existing `@atlas/config` identity/MCP validation helpers rather than creating divergent rules.
</action>

<action>
Add mount runtime default injection so `AtlasMountConfig` maps only to current supported surfaces:

- `identityRoot` seeds `ATLAS_IDENTITY_ROOT` unless explicit `--atlas-identity-root` is present.
- `mcp.name` seeds `ATLAS_MCP_NAME` unless explicit `--atlas-mcp-name` is present.
- `mcp.title` seeds `ATLAS_MCP_TITLE` unless explicit `--atlas-mcp-title` is present.
- `defaults.config` seeds `ATLAS_CONFIG` unless explicit `--config` is present.
- `defaults.cacheDir` seeds `ATLAS_CACHE_DIR` unless runtime env already has `ATLAS_CACHE_DIR` or config overrides it through `defaults.config`.
- `defaults.logLevel` seeds `ATLAS_LOG_LEVEL` unless runtime env already has `ATLAS_LOG_LEVEL` or config overrides it through `defaults.config`.
- `defaults.caCertPath` seeds `ATLAS_CA_CERT_PATH` unless runtime env already has `ATLAS_CA_CERT_PATH`.

Do not add `ATLAS_MCP_RESOURCE_PREFIX` or `--atlas-mcp-resource-prefix`. For `mcp.resourcePrefix`, validate the value using existing MCP identifier validation, but expose it as config-only behavior: it only becomes effective when supplied through existing Atlas config `identity.mcp.resourcePrefix`. If implementation chooses to materialize a temporary config overlay, it must still use existing config schema key `identity.mcp.resourcePrefix` and include tests proving explicit `--config` precedence.
</action>
<acceptance_criteria>
