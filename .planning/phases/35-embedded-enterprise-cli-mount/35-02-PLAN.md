# Plan 35-02: Add Mount Tests Docs And Compatibility Guards

## Goal

Prove enterprise Commander mounting works, standalone Atlas remains compatible, unsupported schema fields stay absent, and docs show one-minute setup using only current supported fields.

## Objective

Executor will add focused mount tests, standalone regression coverage, config-only `mcp.resourcePrefix` documentation/tests, package smoke coverage, and public docs that clearly constrain the schema.

## Scope

- Mounted command behavior tests.
- Wrapper default precedence tests.
- Standalone compatibility regression tests.
- Type/source/doc guards against invented fields.
- Enterprise wrapper documentation.
- Runtime/config docs updates for import path and field semantics.

## Out of Scope

- No implementation of visual branding fields.
- No auth callback/token provider examples.
- No SDK tutorial beyond Commander mount API.
- No new resourcePrefix CLI/env knob.

## must_haves

- Tests cover `attachAtlas(program, { namespace: "acme" })` command mounting.
- Tests cover help for mounted namespace and representative nested subcommand.
- Tests cover wrapper default injection and explicit user flag precedence.
- Tests prove standalone `atlas` help/JSON/error behavior remains compatible.
- Guards fail if public docs/examples mention unsupported fields as wrapper schema.
- Docs show one-minute Commander setup and limits.
