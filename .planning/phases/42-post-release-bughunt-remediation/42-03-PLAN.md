# Plan 42-03: Harden Source Checkout Diagnostics and Store Consistency Footguns

## Goal

Remove lower-level production footguns found by the post-release bug hunt: sparse current checkouts, stale FTS rows, and unclear package dependency boundaries.

## Objective

Executor will add focused tests first, then fix source diagnostics and store consistency behavior without changing normal build semantics.

## Scope

- Detect/report sparse user checkout when `local-git` uses `refMode: current-checkout`.
- Ensure direct section deletion does not leave stale section/chunk FTS rows, or document and enforce structural-only semantics.
- Decide and encode `@atlas/store` Node dependency boundary for `better-sqlite3`.

## Out of Scope

- No CLI onboarding fixes; covered in 42-01.
- No MCP mount/export fixes; covered in 42-02.
- No changes to remote-mode sparse checkout behavior except preserving tests.

## Findings From Bug Hunt

### Current-checkout mode can silently omit docs from sparse working trees

`packages/source-git/src/cache/repo-cache.service.ts` sets sparse checkout result to disabled for `current-checkout` mode without checking actual Git sparse state.

Problem shape:

```ts
const sparseCheckout = { enabled: false, patterns: [] };
event("sparse_checkout_disabled", repo.repoId, git.localPath);
```

If user working tree is sparse, `listMaterializedFiles()` only sees materialized files and docs outside sparse paths disappear from build. Diagnostic says sparse checkout disabled, which is misleading.

### SectionRepository.deleteForDocument leaves stale FTS rows

`packages/store/src/docs/section.repository.ts` deletes chunks/sections only. It does not remove `fts_entries`. Main build path often reindexes later, but direct repository API leaves stale search rows.

### `@atlas/store` Node fallback relies on root `better-sqlite3`

`packages/store/src/db/client.ts` requires `better-sqlite3` when not running under Bun. Root package declares `better-sqlite3`, but `packages/store/package.json` does not. Package is currently private, so this is lower priority.
