# Plan 42-02: Repair Mounted MCP Identity and Public MCP Exports

## Goal

Make enterprise Commander mount MCP identity behave as advertised and expose the complete MCP skill surface from the public package barrel.

## Objective

Executor will add failing tests first, then fix `resourcePrefix` propagation and missing MCP exports.

## Scope

- Support or explicitly reject `AtlasMountConfig.mcp.resourcePrefix`.
- Preferred: plumb mounted `resourcePrefix` through runtime defaults into MCP identity resolution.
- Export first-party skill/use-skill symbols from `@atlas/mcp` barrel.
- Keep standalone CLI setup free of wrapper-only branding prompts.

## Out of Scope

- No changes to skill execution behavior beyond export surface.
- No broader MCP protocol redesign.
- No CLI command context fixes; covered in 42-01.

## Findings From Bug Hunt

### `AtlasMountConfig.mcp.resourcePrefix` accepted then ignored

`apps/cli/src/commander.ts` validates `resourcePrefix` but does not pass it into runtime defaults/config.

Current shape:

```ts
if (config.mcp?.resourcePrefix !== undefined) {
  validateMcpIdentifier(
    config.mcp.resourcePrefix,
    "identity.mcp.resourcePrefix",
  );
}
```

Docs and public type imply this field works. It silently has no effect unless config also sets `identity.mcp.resourcePrefix`.

### MCP public barrel missing use-skill exports

`packages/mcp/src/index.ts` omits symbols that exist internally:

- `USE_SKILL_TOOL`
- `executeUseSkill`
- `registerUseSkillTool`
- `useSkillInputSchema`
- `UseSkillInput`
- `skillArtifactResource`
